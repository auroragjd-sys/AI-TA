# 首屏视觉与商业效果优化方案 (Optimize Landing Page Visuals)

## Why (背景与目的)
目前的着陆页（Landing Page）在视觉上偏向“软萌”，虽然亲和力强，但缺乏作为医疗健康类产品的“专业信任感”和“紧迫感”。
商业宣传上，痛点描述过于平淡，未能直击用户“担心宠物生病却无法判断”的焦虑核心。
首屏布局拥挤，信息层级干扰了核心转化路径。

## What Changes (变更内容)

### 1. 文案优化 (Copywriting)
- **Hero Section**:
    - 主标题保持，副标题从“功能罗列”改为“场景代入”（强调“3秒读懂求救信号”）。
    - 痛点卡片 (`painCards`) 文案调整，强调“省心”、“避坑”和“减少误判焦虑”。
- **行动号召 (CTA)**: 保持清晰，但周围干扰元素减少。

### 2. 视觉风格微调 (Visual Style Refinement)
- **配色**: 在现有暖色基调上，引入少量的 **“科技蓝” (#4A90E2)** 或 **“医疗绿” (#2E7D32)** 作为点缀（如数据图表、图标、强调文字），以中和过度的“软萌”感，提升专业度。
- **装饰元素**: 增加细线条图表、数据分析网格、扫描框等“数据可视化”元素，暗示 AI 算法的精准性。
- **字体**: 优化 `font-family` 定义，确保在 Windows 平台下有合适的圆体回退或优雅的黑体适配，避免风格割裂。
- **对比度**: 调整浅色背景上的低对比度文字（如 `#8e5f4a`），确保符合无障碍阅读标准。

### 3. 布局重构 (Layout Restructuring)
- **Hero 左侧**:
    - **移除** `heroStats`（3种入口、1条闭环...）的三个卡片布局，避免挤占空间。
    - **新增** 在 Hero 下方（Fold 上方或刚好在 Fold 处）的 **Logo Wall / 特性横条**，展示核心能力标签，给 CTA 留出呼吸感。
- **Hero 右侧**:
    - 优化手机样机内的图片展示。不再展示密集的文字列表，而是展示**核心分析结果**（如放大的“情绪热力图”或“健康评分”仪表盘），让用户一眼看到价值。

### 4. 新增模块 (New Modules)
- **信任背书 (Trust Indicators)**: 在 Hero 下方或 Feature 区域前，增加“兽医监制”、“数据来源”或“合作机构”的背书模块（即使是占位符，也要体现结构）。

## Impact (影响范围)
- **受影响文件**:
    - `src/App.tsx`: 主要结构和文案修改。
    - `src/index.css`: 样式、字体定义、动画调整。
    - `src/components/ui/*.tsx`: 可能涉及基础组件样式的微调（如 Card, Button）。
- **受影响功能**: 首屏展示、响应式布局。

## ADDED Requirements (新增需求)
### Requirement: 信任背书模块
系统应在首屏或次屏展示信任背书信息。
#### Scenario: 用户浏览
- **WHEN** 用户滚动过 Hero 区域
- **THEN** 用户应看到“兽医监制”或“权威数据支持”的标识，增强信任感。

## MODIFIED Requirements (修改需求)
### Requirement: Hero 区域布局
Hero 区域应更加简洁，聚焦于“价值主张”和“转化按钮”。
#### Scenario: 首屏加载
- **WHEN** 用户打开页面
- **THEN** 左侧不应出现拥挤的功能卡片，右侧手机样机应清晰展示核心分析图表。

### Requirement: 字体适配
系统应在不同操作系统上保持一致的“圆润”或“专业”风格。
#### Scenario: Windows 用户访问
- **WHEN** 用户使用 Windows 系统访问
- **THEN** 字体不应回退到默认宋体或不协调的黑体，应有指定的备选字体栈。
